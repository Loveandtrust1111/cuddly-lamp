# Contributing to cuddly-lamp

Thank you for your interest in contributing to cuddly-lamp! This document provides guidelines for contributing to the project.

## How to Contribute

### Reporting Issues

- Check if the issue already exists before creating a new one
- Provide clear and detailed information about the issue
- Include steps to reproduce if applicable

### Submitting Pull Requests

1. Fork the repository
2. Create a new branch for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes with clear, descriptive commit messages
4. Test your changes thoroughly
5. Push to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
6. Open a Pull Request with a clear description of your changes

### Code Style

- Write clear, readable code
- Follow existing code conventions in the repository
- Comment complex logic where necessary
- Keep commits focused and atomic

### Pull Request Guidelines

- Provide a clear description of the changes
- Reference any related issues
- Ensure all tests pass (if applicable)
- Update documentation as needed

## Getting Help

If you need help or have questions:
- Open an issue for discussion
- Reach out to the maintainers

## Code of Conduct

Be respectful and constructive in all interactions with the community.

Thank you for contributing!
