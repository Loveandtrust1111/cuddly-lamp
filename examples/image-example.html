This page demonstrates various best practices for using images in web projects.

1. Basic Image with Alt Text

Always include descriptive alt text for accessibility:

<img src="photo.jpg" 
     alt="A scenic mountain landscape at sunset with orange and purple sky"
     width="800"
     height="600">

2. Responsive Images with srcset

Serve different image sizes for different screen sizes:

<img srcset="photo-400.jpg 400w,
             photo-800.jpg 800w,
             photo-1200.jpg 1200w"
     sizes="(max-width: 600px) 400px,
            (max-width: 1000px) 800px,
            1200px"
     src="photo-800.jpg"
     alt="Responsive image example">

3. Modern Image Formats with Fallback

Use modern formats like WebP with fallbacks:

<picture>
    <source srcset="photo.webp" type="image/webp">
    <source srcset="photo.jpg" type="image/jpeg">
    <img src="photo.jpg" alt="Modern format example">
</picture>

4. Lazy Loading Images

Defer loading of off-screen images:

<img src="photo.jpg" 
     alt="Lazy loaded image"
     loading="lazy"
     width="800"
     height="600">

5. Semantic Image with Caption

Use figure and figcaption for images with captions:

<figure>
    <img src="photo.jpg" alt="Mountain landscape">
    <figcaption>Beautiful mountain landscape</figcaption>
</figure>

6. CSS Background Images (Responsive)

Use CSS for decorative images:

.hero {
    background-image: url('photo.jpg');
    background-size: cover;
    background-position: center;
}

7. SVG Icons (Inline)

Use inline SVG for icons that need styling:

<svg width="48" height="48" viewBox="0 0 24 24">
    <circle cx="12" cy="12" r="10"></circle>
    <path d="M12 6v6l4 2"></path>
</svg>

8. Progressive Image Loading

Show a low-quality placeholder while loading:

// JavaScript
const img = document.querySelector('.progressive-image');
const highRes = new Image();
highRes.src = img.dataset.src;
highRes.onload = () => {
    img.src = highRes.src;
    img.classList.add('loaded');
};
